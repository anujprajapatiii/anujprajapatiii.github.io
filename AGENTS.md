# AGENTS.md

Guidance for Claude and other coding agents working in this repository.

## Project Overview

This is Anuj Prajapati's personal portfolio site.

Anuj is a designer with limited coding experience. Explain non-obvious technical
decisions in plain language, and prefer simple, readable solutions over clever
ones. The visual presentation is Anuj's craft: propose design options rather
than inventing visual direction unprompted.

Visual direction lives in `agent-os/strategy.md` and
`agent-os/conventions/styling.md`. Until those are filled in, keep the UI
neutral, minimal, and content-first.

## Start Here

- `agent-os/strategy.md`: project direction, audience, and non-goals
- `agent-os/system-map.md`: operating map for how the site fits together
- `agent-os/conventions/`: architecture, content, and styling conventions
- `agent-os/plans/`: work artifacts for non-trivial changes
- `agent-os/learnings/`: reusable project memory from completed work
- `agent-os/parking-lot.md`: ideas deliberately deferred — do not build these
  unless asked

## Tech Stack

- Framework: Astro v5, static output
- Styling: Tailwind CSS v4 through `@tailwindcss/vite` (no PostCSS config, no
  tailwind.config file)
- Components: shadcn/ui, configured in `components.json`; add primitives with
  `pnpm dlx shadcn@latest add <component>`
- React: only for stateful, client-interactive UI; prefer `.astro` otherwise
- Content: Astro content collections with Zod schemas in `src/content.config.ts`
- Fonts: Geist Sans (body) and Geist Mono (labels/metadata) via Fontsource,
  placeholder until the design token session
- Package manager: pnpm 10+
- Node: v20+
- Deployment: GitHub Pages through GitHub Actions on pushes to `main`

## Commands

```bash
pnpm dev       # local dev server at localhost:4321

```

## Workflow

Use a strategy -> plan -> build -> review -> learn workflow for non-trivial
work.

Small fixes do not need a plan. Work that affects multiple files, public
behavior, content architecture, or deployment gets a plan file in
`agent-os/plans/` first, using `agent-os/plans/README.md` as the template.
Present the plan (especially the Scope section) for approval before building.

At the end of non-trivial work, decide whether to capture a learning note,
update a convention, or update `agent-os/system-map.md`.

When an idea comes up that is out of scope for the current task, add it to
`agent-os/parking-lot.md` instead of building it.

## Core Conventions

- Use `@/` imports for source files. The alias maps to `./src/*`.
- Prefer `.astro` components by default; React only for client interactivity.
- Query content with `getCollection()` from `astro:content`.
- Filter drafts (`data.draft`) before rendering public listings.
- Components use semantic design tokens (`bg-background`, `text-foreground`,
  `text-muted-foreground`, `border-border`, `text-accent`) — never raw hex
  values or one-off colors.
- Site-wide facts (name, nav, links) live in `src/data/site-config.ts`, not
  hardcoded in components.

## Verification

Run the narrowest useful verification for the change:

- `pnpm build` for code, content schema, route, and layout changes
- `pnpm dev` plus a browser check for visual or interactive changes

## Multi-Agent Coordination

- Check `git status --short` before editing.
- Do not overwrite or revert user changes unless explicitly asked.
- Keep edits scoped to the user's request.
- Prefer existing local patterns over introducing new libraries or
  abstractions.
- Update `AGENTS.md` only when durable cross-agent guidance changes.
