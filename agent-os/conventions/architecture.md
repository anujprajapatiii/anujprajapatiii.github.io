# Architecture Conventions

## Rules

- Use `@/` imports for source files. The alias maps to `./src/*`.
- Prefer `.astro` components for static and server-rendered UI.
- Use React only for stateful UI and client interactivity that genuinely
  needs it.
- Content collections are defined in `src/content.config.ts`.
- Query content with `getCollection()` from `astro:content`.
- Filter drafts before rendering public listings.
- Site-wide facts (name, nav, social links) live in `src/data/site-config.ts`.
- Do not edit `dist/` as source. It is generated by `pnpm build`.

## Layout Hierarchy

- `BaseLayout.astro`: the HTML document — head, SEO, theme bootstrapping
- `PageLayout.astro`: the site chrome — header, main slot, footer

Add specialized layouts (e.g. a case-study layout) as children of
`PageLayout` when a content type needs its own shell.
